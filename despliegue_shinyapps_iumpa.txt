GUÍA RÁPIDA PARA DESPLEGAR LA APP SHINY DEL TFG EN SHINYAPPS.IO
================================================================

Objetivo
--------
Subir la carpeta de la aplicación Shiny a shinyapps.io para obtener un enlace público del tipo:

https://TU_USUARIO.shinyapps.io/iumpa-grafos/


0. Comprobación previa
---------------------
El ZIP preparado para despliegue debe contener una carpeta llamada, por ejemplo:

__Interface/

Dentro de esa carpeta debe estar directamente el archivo principal:

__Interface/app.R

Además, dentro de la misma carpeta deben estar todos los CSV y XLSX que lee la aplicación.
No despliegues una carpeta padre que contenga __Interface dentro; despliega la carpeta que contiene directamente app.R.

La versión pública debe usar datos limitados. En concreto, Investigadores_internos.csv no debe incluir la columna Correos.


1. Descomprime el ZIP
--------------------
Descomprime __Interface.zip en una ubicación sencilla de tu ordenador, por ejemplo:

C:/Code/tfg-gcd/__Interface

La estructura debe quedar así:

__Interface/
  app.R
  Investigadores_internos.csv
  Revistas.csv
  es_parte_de_limpio_listas.csv
  ha_publicado_con.csv
  ha_participado_con.csv
  1ha_publicado_en_....csv
  ha_participado_en_....xlsx
  ...


2. Abre RStudio en la carpeta correcta
--------------------------------------
Abre RStudio y carga el archivo app.R.

En la consola de RStudio, comprueba que estás en la carpeta que contiene app.R:

getwd()
list.files()

En list.files() debe aparecer app.R.

Si no estás en la carpeta correcta, puedes fijarla manualmente solo para esta sesión:

setwd("C:/Code/tfg-gcd/__Interface")

No metas ese setwd() dentro de app.R. Es solo para tu consola local.


3. Instala los paquetes necesarios
----------------------------------
Ejecuta una vez en la consola de RStudio:

install.packages(c(
  "shiny",
  "dplyr",
  "tidyr",
  "readr",
  "readxl",
  "echarts4r",
  "rsconnect"
))

Si ya los tienes instalados, no pasa nada.


4. Prueba la app en local
-------------------------
Antes de subir nada, ejecuta:

shiny::runApp(".")

También puedes pulsar el botón Run App de RStudio.

La app debe abrirse y funcionar localmente. Comprueba al menos esto:

- Se cargan los cuatro grafos.
- Funcionan los selectores de investigador y área.
- Funcionan los clics en nodos.
- No aparecen errores de archivos no encontrados.
- No se muestran correos electrónicos.

Si falla en local, no la despliegues todavía. Primero corrige el error local.


5. Crea una cuenta en shinyapps.io
----------------------------------
Entra en shinyapps.io y crea una cuenta.

Después, dentro de shinyapps.io, ve a la zona de tokens o configuración de cuenta.
Allí encontrarás un comando parecido a este:

rsconnect::setAccountInfo(
  name = "TU_USUARIO",
  token = "TU_TOKEN",
  secret = "TU_SECRET"
)

Copia ese comando y ejecútalo en la consola de RStudio.

IMPORTANTE:
- No pegues ese comando dentro de app.R.
- No subas tus tokens a ningún repositorio.
- Es una configuración local de tu cuenta para poder desplegar desde tu ordenador.


6. Despliega la aplicación
--------------------------
Desde la consola de RStudio, ejecuta:

library(rsconnect)

rsconnect::deployApp(
  appDir = "C:/Code/tfg-gcd/__Interface",
  appName = "iumpa-grafos",
  appTitle = "Grafos IUMPA"
)

Cambia la ruta de appDir por la ruta real donde hayas descomprimido la carpeta.

Si ya tienes RStudio situado dentro de la carpeta __Interface, también puedes usar:

rsconnect::deployApp(
  appName = "iumpa-grafos",
  appTitle = "Grafos IUMPA"
)

Durante el despliegue, rsconnect subirá app.R, los datos de la carpeta y las dependencias necesarias.


7. Copia el enlace final
------------------------
Cuando termine correctamente, RStudio/shinyapps.io mostrará una URL similar a:

https://TU_USUARIO.shinyapps.io/iumpa-grafos/

Comprueba que el enlace funciona abriéndolo en el navegador.

Después, usa ese enlace en la memoria:

\url{https://TU_USUARIO.shinyapps.io/iumpa-grafos/}


8. Texto recomendado para la memoria
------------------------------------
En el apartado 7.4 Legado, puedes poner:

Esta aplicación interactiva queda disponible como prototipo asociado al proyecto a través
del siguiente enlace: \url{https://TU_USUARIO.shinyapps.io/iumpa-grafos/}. Su acceso y
reutilización quedan condicionados por la disponibilidad de los ficheros de datos utilizados
y por las consideraciones de protección de datos descritas en esta memoria. La versión
desplegada para consulta utiliza una versión limitada de los datos, excluyendo información
de contacto, como los correos electrónicos, al no ser necesaria para la exploración visual.
De este modo, se reduce la exposición de datos personales no imprescindibles y se mantiene
el carácter exploratorio de la herramienta.


9. Si falla el despliegue
------------------------
Errores frecuentes y solución:

A) Error: app.R no encontrado
   - appDir no apunta a la carpeta correcta.
   - Debes desplegar la carpeta que contiene directamente app.R.

B) Error: archivo CSV/XLSX no encontrado
   - Comprueba que el archivo está dentro de __Interface.
   - Revisa mayúsculas, minúsculas, espacios y acentos.
   - shinyapps.io puede ser más estricto con los nombres de archivo que Windows.

C) Error: paquete no instalado
   - Instala el paquete en local y vuelve a desplegar.
   - Revisa que app.R tenga library(...) para todos los paquetes que usa.

D) Funciona en local pero no online
   - Consulta los logs desde R:

rsconnect::showLogs(
  appName = "iumpa-grafos",
  streaming = TRUE
)

   - También puedes revisar los logs desde el panel web de shinyapps.io.

E) Error por rutas absolutas
   - No uses rutas como C:/Users/...
   - La app debe leer archivos con rutas relativas, por ejemplo:

read.csv2("Investigadores_internos.csv", fileEncoding = "latin1")
readxl::read_excel("ha_participado_en_AlegreGC_FINAL.xlsx")


10. Checklist final antes de poner el enlace en la memoria
----------------------------------------------------------
Antes de considerar cerrado el despliegue, comprueba:

[ ] La app funciona localmente.
[ ] La app funciona desde el enlace online.
[ ] Investigadores_internos.csv no contiene la columna Correos.
[ ] No hay tokens, contraseñas ni setwd() activos dentro de app.R.
[ ] La app no muestra información de contacto.
[ ] Los cuatro grafos cargan correctamente.
[ ] Los filtros funcionan.
[ ] Los paneles informativos funcionan.
[ ] El enlace final está copiado correctamente en la memoria.
